## 什么是【FBIF】OpenRouter

【FBIF】OpenRouter 是一款飞书多维表格的字段捷径插件，它可以让你：

- 在表格中直接调用 AI 大模型（如 GPT-5.2、Gemini 3、Kimi 等）
- 批量处理数据，自动为每一行生成 AI 回复
- 支持深度思考模式，让 AI 进行更深入的推理
- 支持联网搜索，获取最新信息

**适用场景**：
- 批量生成文案、翻译、摘要
- 数据分析和提取
- 问答系统
- 内容审核和分类

## 准备工作

### 第一步：注册 OpenRouter 账号

1. 打开浏览器，访问 [OpenRouter 官网](https://openrouter.ai/)
2. 点击右上角「Sign In」或「Sign Up」
3. 可以使用 Google 账号快速注册

### 第二步：获取 API Key

1. 登录后，点击右上角头像 → 「Settings」
2. 在左侧菜单选择「API Keys」或直接访问 [API Keys 页面](https://openrouter.ai/settings/keys)
3. 点击「Create Key」创建新的 API Key
4. **重要**：复制并妥善保存这个 Key（格式为 `sk-or-v1-...`）

> ⚠️ API Key 只会显示一次，请务必保存好！

### 第三步：充值（可选）

- OpenRouter 提供免费额度，可以先试用
- 如需大量使用，请在 [Billing](https://openrouter.ai/settings/billing) 页面充值
- 支持信用卡、加密货币等支付方式

## 添加捷径到多维表格

### 方法一：从字段捷径市场添加

1. 打开飞书多维表格
2. 点击表格最右侧的「+」号添加新字段
3. 选择「字段捷径」
4. 在搜索框中搜索「FBIF OpenRouter」
5. 找到后点击添加

### 方法二：从已有模板复制

如果你收到了包含此捷径的多维表格模板，可以直接复制使用。

## 配置参数详解

添加捷径后，需要配置以下参数：

### 1. 模型名称（必填）

选择要使用的 AI 模型：

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| **OpenAI GPT-5.2** | 综合能力最强 | 通用场景、复杂任务 |
| **Gemini 3 Flash** | 速度快、价格低 | 简单任务、大批量处理 |
| **Gemini 3 Pro** | 能力强、支持深度思考 | 复杂推理、长文本 |
| **智谱 GLM-4.7** | 中文优化 | 中文内容生成 |
| **Kimi K2.5** | 长文本处理 | 长文档分析 |
| **Kimi K2 Thinking** | 深度推理 | 复杂逻辑问题 |
| **Qwen3 32B** | 性价比高 | 日常使用 |
| **Qwen3 235B** | 超大模型 | 专业任务 |

> 💡 如果以上模型都不满足需求，可选择「其他自定义模型」

### 2. 自定义模型名称（选填）

仅当选择「其他自定义模型」时需要填写。

**格式**：`provider/model-name`

**示例**：
- `openai/gpt-4o` - OpenAI GPT-4o
- `anthropic/claude-4-sonnet` - Claude 4 Sonnet
- `deepseek/deepseek-r1` - DeepSeek R1
- `meta-llama/llama-4-maverick` - Llama 4

> 查看所有可用模型：[OpenRouter 模型列表](https://openrouter.ai/models)

### 3. 提示词 Prompt（必填）

这是发送给 AI 的指令，决定了 AI 会如何处理你的数据。

**基础用法**：直接输入固定文本
```
请将以下内容翻译成英文
```

**进阶用法**：结合其他字段
```
请根据以下产品名称，生成一段 50 字以内的营销文案：{产品名称}
```

**提示词技巧**：
- 明确指定输出格式：「请用 JSON 格式输出」
- 限制输出长度：「请在 100 字以内回答」
- 指定角色：「你是一位专业的翻译专家」

### 4. 深度思考模式（选填）

控制模型是否进行深度推理：

| 选项 | 说明 | 适用场景 |
|------|------|---------|
| **默认** | 模型自动判断 | 大多数情况 |
| **关闭深度思考** | 快速回复，不进行推理 | 简单问题、追求速度 |
| **开启深度思考** | 深入推理后回答 | 复杂问题、数学、逻辑 |

> ⚠️ 深度思考会消耗更多 Token，费用更高

### 5. 启用联网搜索（选填）

| 选项 | 说明 |
|------|------|
| **关闭**（默认） | AI 仅使用训练数据回答 |
| **开启** | AI 会搜索互联网获取最新信息 |

> ⚠️ 联网搜索会产生额外费用（约 $0.002/次）

### 6. OpenRouter API Key（必填）

填入你在准备工作中获取的 API Key。

**格式**：`sk-or-v1-xxxxxxxx...`

> 🔒 API Key 会安全存储，不会泄露

### 7. 最大输出 Token（选填）

限制 AI 生成内容的最大长度。

- **默认值**：4096
- **建议值**：
  - 短回答：512-1024
  - 长文章：4096-8192
  - 深度思考：8192+

### 8. 温度参数（选填）

控制 AI 输出的随机性/创造性。

| 值 | 效果 |
|----|------|
| **0** | 最确定，每次回答几乎相同 |
| **0.7** | 平衡创造性和稳定性（推荐） |
| **1.0** | 默认值 |
| **1.5-2.0** | 更有创意，但可能不稳定 |

## 使用示例

### 示例 1：批量翻译

**场景**：将中文产品描述翻译成英文

| 产品描述（原字段） | AI 翻译（捷径字段） |
|------------------|-------------------|
| 这款手机采用... | This phone uses... |

**配置**：
- 模型：Gemini 3 Flash（速度快）
- 提示词：`请将以下中文翻译成英文，保持专业语调：{产品描述}`

### 示例 2：内容摘要

**场景**：为长文章生成摘要

**配置**：
- 模型：GPT-5.2
- 提示词：`请用 3 个要点总结以下文章的核心内容：{文章内容}`

### 示例 3：数据提取

**场景**：从非结构化文本中提取信息

**配置**：
- 模型：Gemini 3 Pro
- 深度思考：开启
- 提示词：
  ```
  请从以下文本中提取：
  1. 公司名称
  2. 联系人
  3. 电话号码
  
  以 JSON 格式输出，文本如下：{原始文本}
  ```

## 常见问题

### Q: 为什么显示「API Key 无效」？

**可能原因**：
1. API Key 复制不完整
2. API Key 已被删除或过期
3. 存在多余的空格

**解决方法**：重新访问 [API Keys 页面](https://openrouter.ai/settings/keys) 创建新 Key

### Q: 为什么显示「余额不足」？

OpenRouter 账户余额不足，请前往 [Billing 页面](https://openrouter.ai/settings/billing) 充值。

### Q: 为什么输出结果为空？

**可能原因**：
1. 提示词有问题
2. 深度思考模式下 Token 不够
3. 模型暂时不可用

**解决方法**：
- 检查提示词是否正确
- 增加「最大输出 Token」参数
- 更换其他模型尝试

### Q: 如何降低费用？

1. 选择价格更低的模型（如 Gemini 3 Flash）
2. 减少「最大输出 Token」
3. 关闭深度思考和联网搜索
4. 优化提示词，减少输入长度

## 费用说明

### 费用组成

| 项目 | 说明 |
|------|------|
| 输入 Token | 提示词 + 数据消耗的 Token |
| 输出 Token | AI 生成内容消耗的 Token |
| 推理 Token | 深度思考消耗的 Token（如有） |
| 联网搜索 | 每次搜索约 $0.002（如开启） |

### 费用查看

每次调用后，捷径会自动返回：
- 费用($)：本次调用的总费用（美元）
- Token 统计：输入/输出/推理 Token 数量

### 省钱技巧

1. **选择合适的模型**：简单任务用便宜模型
2. **控制输出长度**：设置合理的 max_tokens
3. **按需开启功能**：不需要时关闭深度思考和联网
