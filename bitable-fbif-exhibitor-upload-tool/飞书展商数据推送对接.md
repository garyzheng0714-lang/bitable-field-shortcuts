# 飞书展商数据对接接口技术文档

- **接口版本**：v1.0  
- **更新时间**：2025年7月  
- **文档状态**：正式版  

## 1. 接口概述

本接口用于接收飞书平台推送的展商数据，支持展商信息的创建和更新操作。

### 1.1 接口地址

- **方法**：POST  
- **URL**：`http://service.best-expo.com.cn/geturl/lark/UpExhibitors.ashx`

### 1.2 功能特性

- 支持 JSON 和 Form 两种数据格式
- 自动检测展商是否存在，实现新增/更新操作
- 完整的参数验证和错误处理
- 支持中英文双语言设置
- 展会状态检查，防止向已关闭项目推送数据
- 展位号格式验证，防止特殊字符

## 2. 请求参数

### 2.1 Content-Type

- `application/json`：JSON 格式
- `application/x-www-form-urlencoded`：Form 表单格式

### 2.2 参数列表

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|---|---|---|---|---|
| booth_no | String | 必填 | 展位号，不能包含特殊字符 | A001 |
| hall | String | 必填 | 展馆编号 | W1 |
| company_name | String | 必填 | 公司名称（中文） | 北京科技有限公司 |
| company_name_en | String | 可选 | 公司名称（英文） | Beijing Tech Co., Ltd |
| area | String | 必填 | 展位面积（平方米） | 36 |
| username | String | 必填 | 登录账号（唯一标识） | booth_001 |
| password | String | 必填 | 登录密码 | 123456 |
| booth_type | String | 可选 | 展位类型：RS=光地，SS=标摊，默认 RS | RS |
| contact_name | String | 必填 | 联系人姓名 | 张三 |
| contact_email | String | 必填 | 联系人邮箱 | zhangsan@company.com |
| lang | String | 可选 | 语言设置：CN=中文，EN=英文，默认 CN | CN |

### 2.3 请求示例

**JSON 格式**

```http
POST /GetUrl/lark/UpExhibitors.ashx
Content-Type: application/json

{
  "booth_no": "A001",
  "hall": "W1",
  "company_name": "北京科技有限公司",
  "company_name_en": "Beijing Tech Co., Ltd",
  "area": "36",
  "username": "booth_001",
  "password": "123456",
  "booth_type": "RS",
  "contact_name": "张三",
  "contact_email": "zhangsan@company.com",
  "lang": "CN"
}
```

**Form 表单格式**

```http
POST /GetUrl/lark/UpExhibitors.ashx
Content-Type: application/x-www-form-urlencoded

booth_no=A001&hall=W1&company_name=北京科技有限公司&area=36&username=booth_001&password=123456&booth_type=RS&contact_name=张三&contact_email=zhangsan@company.com&lang=CN
```

## 3. 响应结果

### 3.1 响应格式

所有响应均为 JSON 格式，`Content-Type` 为：`application/json; charset=utf-8`

### 3.2 成功响应

**创建成功**

```json
{
  "code": "200",
  "message": "展商创建成功",
  "booth_no": "A001",
  "username": "booth_001"
}
```

**更新成功**

```json
{
  "code": "200",
  "message": "展商信息更新成功",
  "booth_no": "A001",
  "username": "booth_001"
}
```

### 3.3 错误响应

| 状态码 | 错误类型 | 示例消息 | 说明 |
|---|---|---|---|
| 400 | 参数错误 | 展位号不能为空 | 必填参数缺失或格式错误 |
| 402 | 业务限制 | 项目已关闭 | 展会项目已关闭，不能接收数据 |
| 500 | 系统错误 | 展商创建失败 | 服务器内部错误或数据库操作失败 |

### 3.4 错误响应示例

```json
{
  "code": "400",
  "message": "展位号不能为空"
}
```

## 4. 业务逻辑

### 4.1 数据处理流程

1. **数据接收**：支持 JSON 和 Form 两种格式的自动解析
2. **参数验证**：检查必填字段完整性和格式正确性
3. **项目状态检查**：验证目标展会是否处于开放状态
4. **展商存在性检查**：根据 `username` 查询是否已存在
5. **数据操作**：
   - 如果展商不存在：创建新展商
   - 如果展商已存在：更新展商信息
6. **返回结果**：返回统一格式的 JSON 响应

### 4.2 展位类型转换

| 输入值 | 转换结果 | 说明 |
|---|---|---|
| RS | 光地 | Raw Space - 空地展位 |
| SS | 标摊 | Standard Space - 标准展位 |
| 其他值或空 | 光地 | 默认为光地类型 |

### 4.3 语言设置处理

- EN → 设置为英文
- CN 或其他值 → 设置为中文（默认）

## 6. 故障排查

### 6.1 常见问题

| 问题现象 | 可能原因 | 解决方案 |
|---|---|---|
| 返回 500 错误 | 服务器内部错误 | 联系展商系统供应商 |
| JSON 解析失败 | 请求格式错误 | 验证 Content-Type 和 JSON 格式 |
| 中文乱码 | 编码不匹配 | 确保请求使用 UTF-8 编码 |
